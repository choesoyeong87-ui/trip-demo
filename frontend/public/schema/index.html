ERD

테이블 연관 관계

현재 MVP 기준에서 실제 서비스 흐름에 영향을 주는 관계만 먼저 보여줍니다.

users (1) (N) lodgings

한 명의 판매자(SELLER)는 여러 숙소를 등록할 수 있다

users.user_id = lodgings.seller_id
users (1) (N) bookings

한 명의 회원은 여러 예약을 가질 수 있다

users.user_id = bookings.user_id
lodgings (1) (N) bookings

한 숙소에 여러 예약이 발생할 수 있다

lodgings.lodging_id = bookings.lodging_id
users (1) (N) inquiries

한 명의 회원은 여러 문의를 작성할 수 있다

users.user_id = inquiries.sender_user_id
